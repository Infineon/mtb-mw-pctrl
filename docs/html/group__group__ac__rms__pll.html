AC RMS PLL

General Description

Overview


This block implements Phase-Locked Loops (PLL) for tracking the phase angle of AC signals and calculating real-time RMS values along with the frequency information. The block provides robust phase tracking capabilities for grid-tied power converters and AC measurement applications. All variants are based on the Synchronous Reference Frame Phase-Locked Loop (SRF-PLL) architecture, which uses the Park transformation to project the input signals onto a rotating reference frame and drives the quadrature (or direct) axis component to zero via a PI controller to lock onto the grid phase angle.

Variants and configuration options

This section describes the available variants and configuration options. All variants are based on the SRF-PLL technique and share the same API so that changing the block interface does not require further changes to the application code when switching between the variants.

AC_RMS_PLL_USE_VARIANT_3PH Use the 3-phase variant of ac_rms_pll (default)
AC_RMS_PLL_USE_VARIANT_3PHDD Use the 3-phase double-decoupling variant of ac_rms_pll
AC_RMS_PLL_USE_VARIANT_1PH Use the 1-phase variant of ac_rms_pll
AC_RMS_PLL_USE_INLINE Use function inlining for ac_rms_pll
AC_RMS_PLL_FREQ_FILTER Enable the frequency low-pass filter
AC_RMS_PLL_RMS_FILTER Enable the RMS low-pass filter
ACRMSPLL_DCOFFSET_REJECT Enable DC offset rejection (1-phase only)
ACRMSPLL_3RD_HARMONIC_REJECT Enable 3rd-harmonic rejection (1-phase only)
ACRMSPLL_5TH_HARMONIC_REJECT Enable 5th-harmonic rejection (1-phase only)

AC_RMS_PLL_3PHDD_USE_CTRL_3P3Z_HW Use hardware 3P3Z controller (3PHDD only)


Three-phase variant
This is the default variant that implements the SRF-PLL for three-phase systems using balanced alpha-beta frame inputs. It utilizes the Park transformation for phase detection by minimizing the voltage projected on the quadrature axis. This variant is optimized for three-phase grid applications and provides excellent tracking performance under balanced conditions. The design of this variant is shown in the block diagram below.

The three-phase variant is selected by defining AC_RMS_PLL_USE_VARIANT_3PH in the user config file. This is the default variant.

#define AC_RMS_PLL_USE_VARIANT_3PH

Three-phase double decoupling variant:
This variant extends the SRF-PLL with a Double-Decoupled Synchronous Reference Frame (DDSRF) decoupling network that eliminates double-frequency oscillations during unbalanced grid conditions. This provides superior phase tracking accuracy under voltage sags, single-phase faults, and other grid imbalance conditions.

The DDSRF variant uses two synchronous reference frames (positive and negative sequence) and a cross-decoupling mechanism to separate sequence components, allowing the PLL to track only the positive sequence component even under severe unbalance. The design of this variant is shown in the block diagram below.

Optionally, the PI controller can be replaced with a hardware-accelerated 3P3Z filter by defining AC_RMS_PLL_3PHDD_USE_CTRL_3P3Z_HW. In this case, the ctrl3p3z_ctx field in the configuration must point to an initialized Filter 3p3z instance. See Filter 3p3z for details on configuring the 3P3Z filter. This feature is only available for 3PHDD variant. Support for other variants shall be added in the future.

The three-phase double-decoupling variant is selected by defining AC_RMS_PLL_USE_VARIANT_3PHDD in the user config file.

#define AC_RMS_PLL_USE_VARIANT_3PHDD

Single-phase variant
This variant implements the SRF-PLL for single-phase systems. Since only a single-phase signal is available, a Second Order Generalized Integrator (SOGI) is used to generate the orthogonal component (beta) from the input signal (alpha). The SOGI acts as an Orthogonal Signal Generator (OSG) that creates the missing quadrature signal required for SRF-PLL operation. Once both alpha and beta components are available, the single-phase variant uses the same SRF-PLL architecture as the three-phase variant, providing robust phase tracking capabilities for single-phase grid applications. The design of this variant is shown in the block diagram below.

The single-phase variant is selected by defining AC_RMS_PLL_USE_VARIANT_1PH in the user config file.

#define AC_RMS_PLL_USE_VARIANT_1PH

Additional features available for the single-phase variant

  • DC offset rejection for tracking and removing DC offsets in the input signal
  • 3rd-harmonic rejection for tracking and removing 3rd harmonic distortions
  • 5th-harmonic rejection for tracking and removing 5th harmonic distortions

All single-phase features can be enabled together:

#define AC_RMS_PLL_USE_VARIANT_1PH
#define AC_RMS_PLL_FREQ_FILTER
#define AC_RMS_PLL_RMS_FILTER
#define ACRMSPLL_DCOFFSET_REJECT
#define ACRMSPLL_3RD_HARMONIC_REJECT
#define ACRMSPLL_5TH_HARMONIC_REJECT

Performance optimization
To allow for optimization of execution time, the API functions can be inlined by defining AC_RMS_PLL_USE_INLINE in the user config file. The default is no function inlining.

Using the API
The application must allocate the context structure (ac_rms_pll_context_t) in a memory section of choice. It must set the values of the configuration sub-structure (ac_rms_pll_context_t::config). The static sub-structure (ac_rms_pll_context_t::vars) contains the static memory the block needs to operate. It must not be initialized by the application. The application has to call the block reset function (AcRmsPllReset) to clear the static sub-structure and the block init function (AcRmsPllInit) to initialize the internal configuration.

Example usage of the three-phase variant:

// instantiate an instance of the PLL context
ac_rms_pll_context_t pll_context;
{
// configure common PLL parameters (all variants)
pll_context.config.omega_n = 314.159265f; // 50 Hz nominal frequency in rad/s
pll_context.config.Kp = 267.0f; // PI proportional gain
pll_context.config.Ki = 35530.0f; // PI integral gain
pll_context.config.cterm_min = -10.0f; // lower limit for PI output
pll_context.config.cterm_max = 10.0f; // upper limit for PI output
pll_context.config.dt = 0.0001f; // 10 kHz sampling rate
pll_context.config.lpc_rms_alpha = 0.01570796f; // RMS filter coefficient
pll_context.config.lpc_rms_beta = 0.98429203f; // RMS filter coefficient
pll_context.config.lpc_rms_init = 0.0f; // RMS filter initial value
pll_context.config.lpc_omega_alpha = 0.00942477f; // frequency filter coefficient
pll_context.config.lpc_omega_beta = 0.99057519f; // frequency filter coefficient
pll_context.config.lpc_omega_init = 314.159265f; // frequency filter initial value
#ifdef AC_RMS_PLL_USE_VARIANT_1PH
// configure single-phase variant parameters
pll_context.config.Kosg = 0.707f; // SOGI damping factor
pll_context.config.Krms = 0.707f; // RMS calculation gain
#ifdef ACRMSPLL_DCOFFSET_REJECT
pll_context.config.Kdc = 0.18f; // DC offset compensation gain
pll_context.config.dc_max = 2.0f; // DC offset upper limit
pll_context.config.dc_min = -2.0f; // DC offset lower limit
#endif
#ifdef ACRMSPLL_3RD_HARMONIC_REJECT
pll_context.config.Kosg3 = 0.47140452f; // 3rd harmonic SOGI damping factor
#endif
#ifdef ACRMSPLL_5TH_HARMONIC_REJECT
pll_context.config.Kosg5 = 0.28284269f; // 5th harmonic SOGI damping factor
#endif
#endif /* AC_RMS_PLL_USE_VARIANT_1PH */
#ifdef AC_RMS_PLL_USE_VARIANT_3PHDD
// configure three-phase double-decoupling variant parameters
pll_context.config.lpc_dd_alpha = 0.02221441f; // sequence filter coefficient (alpha)
pll_context.config.lpc_dd_beta = 0.98429203f; // sequence filter coefficient (beta)
pll_context.config.lpc_dd_init = 0.0f; // sequence filter initial value
#ifdef AC_RMS_PLL_3PHDD_USE_CTRL_3P3Z_HW
// hardware 3p3z controller setup
pll_context.config.ctrl3p3z_ctx.vars.variant.base = PPCA_HWFILT3P3Z_SS_0_HWFILT3P3Z0;
pll_context.config.ctrl3p3z_ctx.config.cx_q23[0] = 8388607;
pll_context.config.ctrl3p3z_ctx.config.cx_q23[1] = -8384138;
pll_context.config.ctrl3p3z_ctx.config.cx_q23[2] = 0;
pll_context.config.ctrl3p3z_ctx.config.cx_q23[3] = 0;
pll_context.config.ctrl3p3z_ctx.config.cy_q23[0] = 8388607;
pll_context.config.ctrl3p3z_ctx.config.cy_q23[1] = 0;
pll_context.config.ctrl3p3z_ctx.config.cy_q23[2] = 0;
pll_context.config.ctrl3p3z_ctx.config.limMax_q23 = 8388607;
pll_context.config.ctrl3p3z_ctx.config.limMin_q23 = -8388608;
pll_context.config.ctrl3p3z_ctx.config.gOut = 0;
pll_context.config.scale_3p3z = -266.7151f; // Q0.23 to float scaling factor
Cy_PPCA_HWFILT3P3Z_SS_PeripheralEnable(PPCA_HWFILT3P3Z_SS_0);
#endif
#endif /* AC_RMS_PLL_USE_VARIANT_3PHDD */
// reset and init PLL context instance
AcRmsPllReset(&pll_context);
AcRmsPllInit(&pll_context);
// get some input values (Clarke-transformed alpha/beta)
float32_t alpha = 325.27f;
float32_t beta = 0.0f;
float32_t rms;
// execute PLL function
AcRmsPll(&pll_context, alpha, beta, &rms);
// results available in context:
// pll_context.vars.theta - tracked phase angle
// pll_context.vars.omega_filtered - tracked frequency
// rms - RMS value of input
}
