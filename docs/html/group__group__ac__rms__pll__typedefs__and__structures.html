AC RMS PLL

General Description

Context structure and sub-structures for the AC RMS PLL block.

The context structure contains all the necessary data for AC RMS PLL operation. It is divided into two main parts:

  • Configuration structure (ac_rms_pll_config_t): Must be set by the application
  • Runtime variables structure (ac_rms_pll_static_t): Managed automatically by the library The application must allocate the context structure (ac_rms_pll_context_t) in a memory section of choice. It must set the values of the configuration sub-structure (ac_rms_pll_context_t::config). The static sub-structure (ac_rms_pll_context_t::vars) contains the static memory the block needs to operate. It must not be initialized by the application. The application has to call the block reset function (AcRmsPllReset) to clear the static sub-structure and the block init function (AcRmsPllInit) to initialize the internal configuration.
